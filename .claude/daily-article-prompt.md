# 日次記事生成プロンプト

以下の手順で Zenn の技術記事を1本作成してください。

## ステップ1: pow/spark リポジトリの分析

pow/spark リポジトリを分析して、以下を把握してください：

- 使用している言語・フレームワーク・ライブラリ
- 最近のコミット（直近1〜2週間）で変更された箇所
- package.json, Cargo.toml, go.mod, requirements.txt 等の依存関係
- README やドキュメントに記載された技術的な特徴

## ステップ2: テーマ選定

pow/spark で実際に使われている技術に基づいて、以下の観点からテーマを1つ選んでください：

- 最近のコミットで触れた技術の深掘り
- 使用ライブラリの便利な使い方や落とし穴
- プロジェクトで採用しているアーキテクチャパターンの解説
- 開発中に遭遇しがちな課題とその解決法
- 使用フレームワークの最新アップデート情報

## ステップ3: 既存記事の確認

pow/zenn の `articles/` ディレクトリ内の既存記事を確認し、テーマが重複しないようにしてください。

## ステップ4: 記事作成

1. テーマに基づいた slug を決める（英数字・ハイフン、12〜50文字）
2. 今日の日付（YYYY-MM-DD）を slug の末尾に付ける
3. `articles/{slug}.md` に記事を作成する
4. CLAUDE.md の記事作成ルールに従う

### 記事の構成

```
## はじめに
（pow/spark での実体験に基づく課題や背景を簡潔に）

## 本題
（具体的な手順・コード・解説。pow/spark のコードを参考にしつつ、
 読者が汎用的に使える形で紹介する）

## まとめ
（要点の整理・次のステップ）
```

### 注意事項
- pow/spark のプライベートなコードをそのまま掲載しない
- 技術やパターンを一般化して、読者が自分のプロジェクトに応用できる形にする
- 具体的なコード例やコマンド例を必ず含める

## ステップ5: コミット

- 変更を git commit する
- `published: false` にしておく（手動確認後に公開する）
